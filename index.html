How the Metrics Work

All four metrics measure lexical surface similarity — how much two texts share the same words or character sequences. They do not measure semantic meaning. Scores range from 0 (no overlap) to 1 (identical), displayed here as percentages.

Cosine Similarity Bag-of-words

Treats each text as a word-frequency vector and measures the angle between them. Two texts that use the same words proportionally — even at different lengths — score high.

cos(A, B) = (A · B) / (‖A‖ × ‖B‖)
  1. Tokenize both texts (lowercase, strip punctuation, split on whitespace).
  2. Count word frequencies → two vectors over the shared vocabulary.
  3. Compute dot product of the vectors.
  4. Divide by the product of each vector's Euclidean magnitude.

Word order is ignored. "The cat sat" and "sat the cat" are identical. Length differences are normalized out.

Jaccard Index Set overlap

Measures the proportion of the shared vocabulary relative to the combined vocabulary. Each word is counted once regardless of frequency.

J(A, B) = |A ∩ B| / |A ∪ B|
  1. Tokenize both texts → two sets of unique words.
  2. Count words in both sets (intersection).
  3. Count words in either set (union).
  4. Divide intersection size by union size.

Always ≤ Sørensen–Dice for the same pair — this is a mathematical property, not a bug. Jaccard penalizes dissimilarity more harshly.

Q-gram Similarity (n=3) Character n-grams

Splits each text into overlapping 3-character substrings ("trigrams") and measures their Jaccard overlap. Catches partial word matches and spelling variations that word-level metrics miss.

Q(A, B) = |ngrams(A) ∩ ngrams(B)| / |ngrams(A) ∪ ngrams(B)|
  1. Lowercase both texts, remove punctuation.
  2. Slide a 3-character window across each text → set of trigrams.
  3. Compute Jaccard overlap of the two trigram sets.

"time" and "times" share trigrams "tim" and "ime", so they contribute positively even though they are different words. More robust to morphological variation than word-level metrics.

Sørensen–Dice Coefficient Set overlap

Like Jaccard but weights the intersection double, giving more credit to shared words and producing higher scores for the same overlap. Commonly used in NLP and ecology.

DSC(A, B) = 2|A ∩ B| / (|A| + |B|)
  1. Tokenize both texts → two sets of unique words.
  2. Count the intersection (shared unique words).
  3. Divide 2× the intersection by the sum of both set sizes.

Always ≥ Jaccard for the same pair. If Jaccard = J, then Dice = 2J / (1 + J). Both metrics reach 1.0 only when the two vocabulary sets are identical.

Levenshtein Distance (normalized) Edit distance

Counts the minimum number of single-character edits — insertions, deletions, and substitutions — needed to transform one text into the other. The raw count is normalized to [0, 1] by dividing by the length of the longer text, then inverted so that 1 = identical and 0 = completely different.

Lev(A, B) = 1 − editDistance(A, B) / max(|A|, |B|)
  1. Lowercase both texts.
  2. Run the standard dynamic-programming edit-distance algorithm (two-row DP, O(m×n) time).
  3. Divide the edit count by the length of the longer text.
  4. Subtract from 1 to convert distance → similarity.

Operates on the full raw text — sensitive to punctuation, whitespace, and word order. Unlike the other four metrics, reordering sentences changes the score even if the word sets are identical. Computation time scales with the product of both text lengths, so very long responses will be slower to compute.
